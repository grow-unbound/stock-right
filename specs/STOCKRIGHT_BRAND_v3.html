Brand — Two Token System
Amber physics constraint: The amber hue (#C8712A) has luminance 0.2426. No background — including pure white — can provide 7:1 contrast against it. This is a spectral property of orange-amber. Solution: two tokens — brand-ui for non-text elements (buttons, icons), brand-text for labels and smaller elements.

05 — Navigation · Responsive Pattern
Bottom Tabs (Mobile) + Side Nav (Desktop)

One canonical navigation system, two responsive layouts. Mobile (<640px): bottom tab bar with FAB. Desktop (>640px): persistent side nav + top bar. No top tabs — they conflict with the side nav and create duplicate navigation patterns.

Navigation rule: The dashboard in previous versions showed bottom tabs + side nav + top tabs simultaneously. This was reference-only. In production, only ONE navigation system is active per breakpoint. Implement with @media (max-width: 639px) to toggle.

08 — Brand Language
Voice, Tone & Terminology
🧓
Trusted Elder
Speak like an experienced mandi accountant — calm, clear, never condescending. Every message assumes the user is busy, hands-on, and not looking at a screen. Get to the point in one line.
✓ Do
240 bags recorded
✗ Don't
Transaction committed successfully
📒
Physical Familiarity
The app should feel like a smarter version of their existing paper register. Warm backgrounds, serif numbers, ink-like dark text. If it feels like a bank app, we've gone wrong.
✓ Do
Warm cream bg, Noto Serif numbers
✗ Don't
Pure white + blue highlights
🚦
Amber = Action
Brand amber is used exclusively for things that need user action or indicate current state. Never decorative. When they see amber, they know to tap or pay attention.
✓ Do
CTAs, active nav, focus rings
✗ Don't
Section headers, decorative borders
📶
Offline Transparency
Never hide the network state. Users in rural areas expect outages. Show queue count prominently. "3 entries queued" builds trust. Silent sync builds anxiety.
✓ Do
⚡ 3 queued · syncing
✗ Don't
Spinner with no explanation
🌐
Mother Tongue First
Default to the user's language on first open. Telugu in Andhra/Telangana, Hindi in UP/Bihar, Tamil in TN. English is the fallback, not the default. UI labels must be translatable — no hardcoded English strings.
✓ Do
సరుకు లోపలికి / माल अंदर
✗ Don't
Record Inward (only)
🇮🇳
Indian Numerics
Always use Indian number formatting. ₹2,50,000 not ₹250,000. "2.5 Lakh" not "250K". Dates as DD/MM/YYYY. These are not preferences — they are the user's native format.
✓ Do
₹2,47,500 · 07/05/2026
✗ Don't
₹247500 · 2026-05-07
Terminology — domain language only
✓ Use these words
Inward — not Receiving
Outward — not Shipping / Dispatch
Party / Customer — not Client
Lot / Batch — not SKU
Godown / Warehouse — both OK
Bags — not Units
Collection — not Payment / Settlement
✗ Never use
SKU, ASN, BOL, WMS, ERP
Optimize, synergize, leverage
Transaction committed
Sync completed
Record not found
HTTP error / server unavailable
Session expired (say: please log in again)

✓ Always

  • Warm white/cream backgrounds — never pure clinical white as the only bg
  • Noto Serif for all headings and numbers — establishes register-like trust
  • brand-ui (#C8712A) only for non-text: button bg, icon fill, active indicator
  • brand-text (#8C4A12) for any amber-colored text or label
  • Input font-size: 16px — prevents iOS zoom on focus
  • Indian number format: ₹2,50,000, DD/MM/YYYY
  • 48px tap zone on all interactive elements (hitSlop in RN)
  • Dark mode: soil text (#1C1A16) on amber CTA — not white
  • Skeleton screens during loading — never bare spinners
  • Show offline queue count visibly in header/topbar
  • Bottom tabs on mobile, side nav on desktop — never both visible

✗ Never

  • No pure white (#FFF) as page background in light mode — use #FEFCF8
  • No dark nav bars in light mode
  • No white text on amber in any context — it fails WCAG
  • No purple, blue-grey, or cool-tone accents
  • No decorative or illustrative use of brand amber
  • No Inter, Roboto, or system fonts for headings
  • No Latin-only serif (Lora, Playfair) — breaks Indian script rendering
  • No silent failures — every action gets visual + haptic feedback
  • No error codes or stack traces to end users
  • No top tab navigation — conflicts with side nav pattern
  • No UI that requires more than 3 taps for inward/outward entry
